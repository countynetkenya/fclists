FCLists — User Guide

FCLists adds the dense, computed lists a plain Frappe List View can't give you: live on-hand stock, reorder gaps, batch expiry, QuickBooks-POS-style sales / payment histories, AR & AP balances, aging buckets, live account balances and best-seller rankings — plus a ready-made dashboard, charts and KPI cards the moment you install it. Every report is a native ERPNext Script Report over your own data, and the dashboard is built on plain frappe dashboards — nothing is stored twice, no extra BI tool is needed, and each one obeys the permissions you already have.

You see a report only if you already hold the role for it. FCLists creates no new roles. The stock boards want a Stock role (Stock User / Stock Manager); the transaction histories want an Accounts role (Accounts User / Accounts Manager). If a report isn't in your report list, you're missing its native role — ask your admin.

1. "I want to do X" → open this list

Reach any of these from Reports (search the report name in the awesomebar), or from the jump button FCLists adds to the matching desk List View (see §3).

You want to…OpenWhich gives you

2. Filter, then read

Each report opens with sensible filters at the top — a date range, a customer, an item, a warehouse, or a "last N days" window, depending on the report. Set them and the table refreshes. Two habits worth keeping:

  • Narrow the date range first. The transaction histories default to a recent window so they stay fast; widen it only when you need older records.
  • Export from the report menu (the ⋮ menu → Export) when you want the numbers in a spreadsheet. FCLists doesn't rebuild export — it's the native desk feature.

4. Your dashboard, charts and KPI cards

FCLists installs a ready-made dashboard so you get an at-a-glance overview the moment the app goes on — no Frappe Insights or other BI tool needed. It's all built on the plain frappe dashboard, chart and number-card features every ERPNext site already has.

You want to…Open / addWhich gives you
See the whole picture in one screen Dashboard → FCLists BI A pre-built dashboard laying out the sales trend, top customers, and the KPI cards below. Set it as your home dashboard or pin it.
Watch the sales trend / your biggest customers Charts FClist Sales Trend, FClist Top Customers A monthly sales line over the last year, and a bar chart of your ten highest-billed customers.
Keep a KPI in the corner of your eye Number Cards (add to any workspace) Five one-number glances: Items Below Reorder, Batches Expiring ≤30d, Overdue Invoices, Total Sales MTD and Overdue AR.

The cards and charts are role-safe: the stock cards want a Stock role, the money cards want an Accounts role, so a card never shows a figure to someone who couldn't already see it. To place a card, open a Workspace or Dashboard, choose Edit → Add Card, and pick any card named FClist…. If you also run Frappe Insights, these same lists make a natural data source — but FCLists works fully without it.

5. Why the numbers are always live (and always yours)

  • Computed, not copied. On-hand qty, margin, days-to-expiry and tender splits are calculated from your native tables (Bin, Stock Ledger Entry, Batch, the invoice payment rows) each time you run the report — never from a second copy that could drift.
  • Your permissions apply automatically. The reports read through the ORM, so any User Permissions you have (e.g. restricted to one warehouse or company) filter the rows for you without any extra setup.
  • Role-gated, never public. Each report is limited to its native roles, so a report never shows figures to someone who couldn't already see them in ERPNext.

6. If a report is empty or missing

  • Missing from your report list? You don't hold its native role. Stock boards need a Stock role; histories need an Accounts role. Ask your admin to grant it.
  • Runs but shows nothing? Your filters (usually the date range or warehouse) exclude everything, or your User Permissions restrict you to records that have no matching data. Widen the date range or clear a filter.
  • Whole app quiet? An admin can switch FCLists off per-site with fclists_enabled: false in site_config.json (it defaults to on). If every report is empty, check that flag first.
