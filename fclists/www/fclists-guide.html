FCLists — User Guide

FCLists adds the dense, computed lists a plain Frappe List View can't give you: live on-hand stock, reorder gaps, batch expiry, and QuickBooks-POS-style sales / payment histories. Every one is a native ERPNext Script Report over your own data — nothing is stored twice, and each report obeys the permissions you already have.

You see a report only if you already hold the role for it. FCLists creates no new roles. The stock boards want a Stock role (Stock User / Stock Manager); the transaction histories want an Accounts role (Accounts User / Accounts Manager). If a report isn't in your report list, you're missing its native role — ask your admin.

1. "I want to do X" → open this list

Reach any of these from Reports (search the report name in the awesomebar), or from the jump button FCLists adds to the matching desk List View (see §3).

You want to…OpenWhich gives you
See what's actually on the shelf, its cost, its price and its margin FClist Item Stock Per item: on-hand qty (summed across warehouses), valuation, selling price, margin & margin%, reorder level, and units sold in the last N days (velocity).
Know what to re-order before you run out FClist Reorder Only items whose on-hand has fallen to or below their reorder level, with the shortfall and the default supplier.
Clear near-expiry stock first (FEFO) FClist Batch Expiry Every batch with an expiry date, sorted earliest-first: days-to-expiry, qty remaining, and an Expired / Expiring / OK status flag.
See exactly what moved in and out of stock FClist Stock Movement Recent Stock Ledger moves with the signed qty split into clear in / out columns, plus the voucher that caused each one.
Reprint or look up a sale (end-of-day register) FClist Sales History A chronological register of every sales receipt with a link to open/print it, filterable by date, customer and the cashier who rang it.
See which invoices are still owed / overdue FClist Sales Invoice Every sales invoice with total, outstanding, status and a computed Overdue flag (past due date and still owing).
Reconcile a till's takings by tender FClist POS Invoice Every POS receipt with its payment split (e.g. Cash: 500, M-Pesa: 1200) and a return flag.
Audit refunds / credit notes FClist Returns Every return (credit note): date, customer, the invoice it was returned against, and amount.
Match the day's collections to the bank / M-Pesa statement FClist Payments Every incoming payment: party, amount, mode of payment, and the bank / M-Pesa reference.

2. Filter, then read

Each report opens with sensible filters at the top — a date range, a customer, an item, a warehouse, or a "last N days" window, depending on the report. Set them and the table refreshes. Two habits worth keeping:

  • Narrow the date range first. The transaction histories default to a recent window so they stay fast; widen it only when you need older records.
  • Export from the report menu (the ⋮ menu → Export) when you want the numbers in a spreadsheet. FCLists doesn't rebuild export — it's the native desk feature.

3. The jump buttons on your List Views

FCLists also enriches four built-in desk List Views. Open one and you'll find an extra button that takes you straight to the matching dense board — no need to hunt for the report:

On this List ViewYou get
ItemA Disabled indicator + a Stock Board button → FClist Item Stock.
BatchExpired / Expiring colour indicators + an Expiry Board button → FClist Batch Expiry.
Sales InvoiceOverdue / Unpaid indicators + a Sales History button → FClist Sales History.
POS InvoiceA Return indicator + a POS Board button → FClist POS Invoice.

These additions are on top of whatever ERPNext already shows on those lists — FCLists extends the native list settings, it never replaces them, so nothing you had before is lost.

4. Why the numbers are always live (and always yours)

  • Computed, not copied. On-hand qty, margin, days-to-expiry and tender splits are calculated from your native tables (Bin, Stock Ledger Entry, Batch, the invoice payment rows) each time you run the report — never from a second copy that could drift.
  • Your permissions apply automatically. The reports read through the ORM, so any User Permissions you have (e.g. restricted to one warehouse or company) filter the rows for you without any extra setup.
  • Role-gated, never public. Each report is limited to its native roles, so a report never shows figures to someone who couldn't already see them in ERPNext.

5. If a report is empty or missing

  • Missing from your report list? You don't hold its native role. Stock boards need a Stock role; histories need an Accounts role. Ask your admin to grant it.
  • Runs but shows nothing? Your filters (usually the date range or warehouse) exclude everything, or your User Permissions restrict you to records that have no matching data. Widen the date range or clear a filter.
  • Whole app quiet? An admin can switch FCLists off per-site with fclists_enabled: false in site_config.json (it defaults to on). If every report is empty, check that flag first.
